# Satisfactory 製造ライン計算ツールを React + Vite で実装してください

## 概要

Satisfactory の製造ラインを視覚的に構築・計算できる Web アプリを作成してください。

「作りたいアイテムを何個/分で生産したいか」を入力すると、必要な中間素材・設備・設備台数・採掘量を逆算して表示します。

UI は「カード + 接続線」で構成されたファクトリー図のようにしてください。

---

# 技術要件

* React
* Vite
* TypeScript
* Zustand または Context API で状態管理
* カード接続UIは React Flow を使用
* スタイリングは Tailwind CSS
* 完全フロントエンドのみ
* 静的ホスティング可能な構成
* データは JSON で管理

---

# UI仕様

## 基本構成

画面は以下で構成してください。

* 左サイドバー

  * アイテム検索
  * 生産目標設定

# ノード（カード）仕様

各カードは1工程を表す。

例:

* 鉄インゴット精錬
* 鉄板製造
* ネジ製造

カードには以下を表示。

## ヘッダー

* 製造機名

  * 例:

    * 精錬炉
    * 製作機
    * 組立機

* レシピ名

* オーバークロック率

## 本文

入力素材:

* アイテム名
* 消費量/min

出力素材:

* アイテム名
* 生成量/min

設備情報:

* 必要台数
* 1台あたり効率
* 消費電力

---

# 接続線仕様

* 素材の流れをエッジで表現
* エッジラベルに throughput 表示

  * 例:

    * 60/min
    * 240/min
* 色分け対応
* 将来的に Conveyor Mk. を表示しやすい構成

---

# 生産計算仕様

## 入力

ユーザーは以下を入力可能。

* 作りたいアイテム
* 生成量/min

例:

* Iron Plate
* 40/min

---

# 逆算ロジック

入力から再帰的に必要素材を逆算してください。

例:

40 Iron Plate/min を作る場合:

Iron Plate Recipe:

* 30 Iron Ingot -> 20 Iron Plate

必要:

* Iron Ingot 60/min

Iron Ingot Recipe:

* 30 Iron Ore -> 30 Iron Ingot

必要:

* Iron Ore 60/min

設備:

* 製作機 2台
* 精錬炉 2台

もしくは:

* オーバークロック 200% で 1台

---

# オーバークロック仕様

設備ごとにクロック率を変更可能。

* 1%〜250%
* クロック率で生産量変化
* 必要台数再計算
* 電力消費再計算

---

# レシピ仕様

JSONで管理。

構造例:

```json
{
  "id": "iron_plate",
  "name": "Iron Plate",
  "machine": "Constructor",
  "duration": 6,
  "inputs": [
    {
      "item": "Iron Ingot",
      "amount": 3
    }
  ],
  "outputs": [
    {
      "item": "Iron Plate",
      "amount": 2
    }
  ]
}
```

---

# 計算ルール

必要生産量/min を基準に:

```ts
recipeOutputPerMin =
(outputAmount / duration) * 60

machinesNeeded =
targetRate / recipeOutputPerMin
```

オーバークロック適用時:

```ts
effectiveRate =
baseRate * (clockSpeed / 100)
```

* 素材依存関係でエッジ接続
